Endpoints:
GET <http://localhost:8000>/api/hotels = Получение списка всех отелей.
GET <http://localhost:8000>/api/hotels/<hotel_id> = Получение информации о конкретном отеле.
POST (...)/api/bookings = Бронирование отеля.
GET (...)/api/reviews = Получение отзывов о конкретном отеле.
POST (...)/api/reviews = Оставление отзыва о отеле.


Главным файлом для взаимодействия с проектом является manage.py:
  В теминале/командной строке (в нужной директории) server запускается командой "python manage.py runserver 8000" [8000 - порт]

"Архитектура":
В папке djangoTest/backend основная логика:
  В products/models.py сформированы модели объектов, предствленных в БД (Отели, Отзывы и Бронирования)
  В api/views.py логика обработки запросов
  В api/urls.py указана связь запросов и методов из views.py, для соответствующей обработки
  В basicproj/ распределение urls и settings


В папке djangoTest/req_client находится (depricated) базовый скрипт для взаимодействия с сервером через питон (а не только postman)
